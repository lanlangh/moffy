# アートアセット仕様（本番イラストの用意と組み込み）

コンセプト画像（リポジトリ直下の `ChatGPT Image 2026年6月19日 *.png` 5枚）の世界観を、
アプリ内に**実イラストとして**反映するための仕様。オーナーが個別画像を生成 → 開発が
`Image.asset` で組み込む、という分担で進める。

> 現状: 卵は暫定のベクター描画（`lib/core/widgets/egg_art.dart`）。本アセットが揃い次第、
> `EggArt` の利用箇所を `Image.asset(...)` に置き換える（差し替え点は卵=1ウィジェットに集約済み）。

---

## 1. 共通仕様（全アセット）

- **形式**: PNG、**背景透過**（必須）。余白は被写体の周囲に均等に少しだけ。
- **解像度**: 正方形キャンバス。卵・キャラは **1024×1024 px**（高解像度1枚でOK。Flutter が縮小）。
  - 端末解像度別（@2x/@3x）を分けなくてよい（単一高解像度を `Image.asset` で表示）。
- **トーン**: コンセプト準拠＝やわらかい3D風・パステル・あたたかい。影は黒でなく茶系。
- **命名**: 半角英小文字・スネークケース。下記の正確な名前で（コードが参照する）。
- **配置先**: 後述の `assets/images/...` フォルダにそのまま入れる。

---

## 2. 優先度1：卵（最優先・オーナーの最重要指摘）

卵は **レアリティ4種 × 成長3段階 = 12枚**。レアリティ色はデザイントークンに準拠：
common=灰緑 / rare=青 / sr=紫 / ssr=金。

| レアリティ | 段階 | ファイル名 | 備考 |
|---|---|---|---|
| common | 無傷 | `egg_common_0.png` | 孵化進捗 0〜19% |
| common | ヒビ小 | `egg_common_1.png` | 20〜49% |
| common | ヒビ大 | `egg_common_2.png` | 50〜99%（孵化直前） |
| rare | 無傷/小/大 | `egg_rare_0.png` / `_1` / `_2` | 同上 |
| sr | 無傷/小/大 | `egg_sr_0.png` / `_1` / `_2` | 同上 |
| ssr | 無傷/小/大 | `egg_ssr_0.png` / `_1` / `_2` | きらめき入りでも可 |

- **巣は含めない**（巣はアプリ側の「巣リング」が描く）。卵単体を中央に、底が少し平らだと収まりが良い。
- まず **common の3枚**だけでも先に入れれば、暫定描画から実イラストに切り替えられる。

配置先: `assets/images/egg/`

---

## 3. 優先度2：Mofi（キャラ・図鑑/孵化で使用）

MVPは **15種 × 2（通常・色違い）= 30枚**（PRD §5 図鑑30）。3系統：
- スライム系（例: ぷにモフィ/ぷるモフィ/キングスライム/エンペラスライム）
- どうぶつ系（例: モフハム/フェネモフィ/フワリス/モフキング）
- ドラゴン系（例: ベビードラ/ホカドラ/フレイムドラ/エンシェントラン）

| 種別 | ファイル名 | 備考 |
|---|---|---|
| 通常 | `mofi_{species}.png` | 例: `mofi_punimofi.png` |
| 色違い | `mofi_{species}_shiny.png` | S13 色違い（配色違い） |

- 正面・全身・かわいいポーズ。図鑑サムネ＆孵化演出で使う。
- 種スラッグ（`{species}`）の確定リストは別途すり合わせ（コードの図鑑エントリ定義と一致させる）。

配置先: `assets/images/mofi/`

---

## 4. 優先度3：UI/通貨/バッジ（任意・後回し可）

| 用途 | ファイル名 | 備考 |
|---|---|---|
| ポイント | `ui_point.png` | 通貨バッジ用（無くてもアイコンで代替中） |
| ジェム | `ui_gem.png` | 〃 |
| レアR/SR/SSR | `badge_r.png` / `badge_sr.png` / `badge_ssr.png` | 現状はコードのバッジで代替可 |

配置先: `assets/images/ui/`

---

## 5. 組み込み手順（開発側・アセットが届いたら）

1. 上記フォルダに画像を入れる。
2. `pubspec.yaml` の `flutter:` に次を追加（**ファイルが存在してから**追加する／空ディレクトリ宣言は `flutter pub get` が失敗する）:
   ```yaml
   flutter:
     uses-material-design: true
     assets:
       - assets/images/egg/
       - assets/images/mofi/
       - assets/images/ui/
   ```
3. `EggArt`（`lib/core/widgets/egg_art.dart`）を `Image.asset('assets/images/egg/egg_${rarity}_${stage}.png')` ベースに置換（暫定描画はフォールバックとして残せる）。
4. 図鑑・孵化演出の Mofi も `Image.asset('assets/images/mofi/...')` に。
5. iOS は **配信前に同梱必須**（google_fonts はネット取得だが画像は同梱が安全）。CI で `flutter analyze`／実機で表示確認。

---

## 6. まず最初の一歩（最小で効果を出す）

**`egg_common_0/1/2.png`（無傷・ヒビ小・ヒビ大）の3枚**を生成 → 入れてもらえれば、
ホームの主役卵を実イラストに切り替えます（暫定ベクターから即アップグレード）。
それで手応えを見てから、レア卵→キャラ、と広げるのが安全で速いです。
