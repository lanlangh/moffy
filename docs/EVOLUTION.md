# Mofi 進化 仕様（v1.0 / 2026-07-02 オーナー決定）

> 決定: **重複入手で進化・2段（ベビー→アダルト）**。設計フォーク2点はオーナー承認済み。

## 1. コンセプト
「**同じ Mofi を重ねて育てると進化する**」。孵化して図鑑を集めるコレクション型の手触りに
そのまま乗せ、**重複（ダブり）に“進化”という価値**を与える（＝リテンション／課金導線）。

- 孵化フロー自体は不変: SNS削減 → pt → 卵が育つ（ヒビ）→ 孵化 → Mofi が図鑑に登録。
- 追加要素: 図鑑に登録済みの同一 Mofi を**規定数そろえると、その姿がアダルトに進化**。

## 2. トリガー = 重複入手数（`obtainedCount`）
- 図鑑エントリ（`MofiDexEntry`）は既に **同一個体の累計入手数 `obtainedCount`** を持つ
  （`uq_collection_dex` / `fn_hatch_egg` がサーバーで加算・クライアントは読み取り専用）。
- 進化段階は **`obtainedCount` としきい値から算出するだけ**（新規の書き込み経路を作らない）。
- **改ざん耐性**: `obtainedCount` はサーバー専管（RLSでクライアント書込を封鎖済み）。段階は
  その読み取り値から導出するので、**進化段階を偽装できない**（経済セキュリティ監査の前提を維持）。

## 3. 段階（2段）
| 段階 | 名称 | 条件（既定） |
|---|---|---|
| 1 | ベビー | 初回孵化（`obtainedCount >= 1`） |
| 2 | アダルト | 同一個体を規定数そろえる（既定 `obtainedCount >= 3`） |

- しきい値（既定 3）は **`app_config` の SSOT**（`mofi_evolve_stage2` 等）に置き、直書きしない。
  レア度別に変える余地あり（例: N/R=3・SR/SSR=2）。まずは一律3で開始。
- 3段目は将来拡張（`app_config` にしきい値を足すだけで拡張可能な作りにする）。

## 4. 色違い（shiny）
- **別イラスト不要**。各段階の**色味をコードで変える**（`RarityToken` の色相をずらす等）。

## 5. 図鑑への影響
- **エントリ総数は 30 のまま**（15種 × 通常/色違い）。増やさない。
- 各マスに**現在の進化段階の姿**を表示（`obtainedCount` で自動的にアダルト表示へ）。
- 詳細シートに「**あと ○ 体で進化**」＋進化ライン（ベビー→アダルト）を表示。

## 6. 演出
- 孵化時、その孵化で `obtainedCount` がしきい値を跨いだら **「進化！」演出**（既存 `hatch_overlay`
  の光演出を流用。通常孵化は「誕生」、進化到達は「進化」に出し分け）。

## 7. 実装インパクト（小さめ・経済RPCは作り直さない）
- **DB**: `app_config` に進化しきい値を1〜数行追加するのみ（経済RPC・報酬経路は不変）。
- **ドメイン**: `MofiDexEntry` に `evolutionStage`（`obtainedCount` としきい値から算出）を追加。
- **表示**: `MofiSubject` を `Image.asset('assets/images/mofi/mofi_<id>_<stage>.png')` ＋
  **errorBuilder で現行の種族アイコンにフォールバック**（＝アート未着でも動く・届いた分から差し替わる）。
- **モック**: プレビュー用に一部エントリの `obtainedCount` を 3+ に仕込み、進化状態を確認可能にする。
- **演出**: `fn_hatch_egg` の戻り（`obtainedCount`/新規登録か）から「進化到達」を判定して出し分け。

## 8. 必要アート
`docs/IMAGE_ASSETS_NEEDED.md` を参照（Mofi 15種 × 2段 = 30体 ＋ 卵4種）。
