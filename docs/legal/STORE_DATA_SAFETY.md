### 2-3. `PACKAGE_USAGE_STATS`（機微権限）の宣言根拠 — 最重要

Google Play では `PACKAGE_USAGE_STATS` のような機微権限・特別なアプリアクセスについて、**コア機能との直接的関連の説明**と**プライバシーポリシーでの利用目的明示**が必須です。Moffy の宣言根拠は以下：

- **権限**：`android.permission.PACKAGE_USAGE_STATS`（OS設定「使用状況へのアクセス」をユーザーが手動許可）
- **コア機能との関連**：本アプリの中核は「SNS利用時間の削減量をポイント化してキャラクターを育成する」こと。利用時間の取得は**この中核機能の提供に不可欠**であり、代替手段が存在しない。
- **取得範囲の限定**：対象4SNSのみ集計。他アプリの利用状況は集計・送信しない。
- **目的外利用の否定**：取得データを広告・第三者販売・プロファイリングには使用しない。
- **プラポリ整合**：`privacy_policy.md` §1-1 に同一内容を記載済み（申告と一致）。
- **権限なしでも動作**：未許可でもアプリは利用可能（クラッシュさせない）。ポイント計算のみ停止し再許可導線を提示（ARCHITECTURE §4-5）。

> 提出時：Play Console の「権限の宣言（Permissions declaration）」および「データ安全性」両方で、上記の用途を整合的に記述すること。スクショ・ストア説明文でも用途を誤認させない（例「あなたのSNS利用を監視」等の不安を煽る表現は避ける）。

## 3. App Store「プライバシー栄養ラベル」対応表（iOS追従時用）

> iOS版は v1.1 で追従予定。iOSでは利用時間取得に Screen Time / DeviceActivity（FamilyControls Entitlement・Apple承認制）を用いる想定。以下は App Store Connect の「App のプライバシー」設定の対応表。

| ラベル区分 | データタイプ | 該当 | リンク（収集データを本人IDに紐づけるか） | トラッキングに使用 |
|---|---|---|---|---|
| **Usage Data（使用状況データ）** | Product Interaction（製品操作） | あり（PostHog） | ユーザーに紐づく（Linked） | いいえ |
| **Diagnostics（診断）** | Crash Data / Performance Data | あり（Sentry） | ユーザーに紐づかない運用を基本（Not Linked） | いいえ |
| **Identifiers（識別子）** | User ID | あり（アカウント識別子） | ユーザーに紐づく（Linked） | いいえ |
| **Contact Info（連絡先情報）** | Email Address | 連携時のみあり | ユーザーに紐づく（Linked） | いいえ |
| **Purchases（購入）** | Purchase History | あり（購読状態。RevenueCat） | ユーザーに紐づく（Linked） | いいえ |
| **Usage Data（その他）** | Other Usage Data（SNS利用時間） | あり（アプリ機能のため） | ユーザーに紐づく（Linked） | いいえ |
| **Health & Fitness / Location / Contacts / Browsing History 等** | — | **なし** | — | — |
| **App Tracking Transparency（ATT）** | トラッキング | **行わない**（広告・データブローカー連携なし） | — | **トラッキングなし** |

iOS追従時の必須対応：
- **Sign in with Apple の提供**：他社ソーシャルログイン（Google等）を提供する場合、Apple のサインインも提供（Guideline 4.8 / 5.1）。
- Screen Time / FamilyControls 利用目的を App 内およびプラポリで明示。
- 上記ラベルは iOS実装の確定後に最終確認（実装と一致させる）。

## 4. 景品表示法・断定/誇大表現チェック

ストア説明文・スクショ・アプリ内コピー・課金画面で、以下を厳守する。

### 4-1. 禁止表現（使用してはならない）

| NG表現 | 理由 | 推奨言い換え |
|---|---|---|
| 「100%安全」「完全に保護」 | 断定・優良誤認 | 「業界標準のセキュリティ対策を実施」 |
| 「絶対に時間が減る」「必ず痩せる/集中力が上がる」 | 効果保証・根拠なき断定 | 「時間の削減を楽しく続けやすくする」 |
| 根拠のない実績数字（「利用者の90%が改善」等） | 不当表示（裏付けなし） | 実測・出典のある数値のみ。なければ記載しない |
| 「業界No.1」「最も効果的」 | 優良誤認（裏付けなし） | 客観的根拠がなければ使わない |

### 4-2. 価格・サブスク表示の一致チェック（景表法＋ストア審査 3.1.2 / 課金規則）

宣伝・課金画面の数値は、実装値（`lib/core/constants/pricing.dart` / `docs/PRICING.md`）と**一字一句一致**させる。

| 確認項目 | 確定値 | チェック |
|---|---|---|
| 月額 | 480円/月（税込） | [ ] 説明文・課金画面・特商法が一致 |
| 年額 | 4,800円/年（税込、約17%OFF、月あたり約400円） | [ ] 実請求額（4,800円/年）を主表示・月割は従属表示 |
| 無料トライアル | 7日間 | [ ] 「7日間無料、その後 480円/月（または4,800円/年）で自動更新」と明示 |
| 自動更新 | 24時間前まで未解約で自動更新 | [ ] 課金画面・規約・特商法で明示 |
| 解約方法 | 各ストアの定期購読管理（アプリ内不可） | [ ] 課金画面に導線/案内 |
| 各種URL | プラポリ/利用規約/特商法のURL | [ ] 課金画面・ストア掲載に掲示 |

### 4-3. 未実装機能を売らない（誇大表示の予防）

- **詳細分析（曜日/時間帯/SNS別/月次/予測）は v1.1送りで未実装** → 課金画面・ストア説明で**プレミアム特典として宣伝しない**（PRICING §2）。
- **広告削除**：MVPは広告SDK未導入のため、「広告削除」を強く前面に出さない（広告が無い間は控えめ表現に留め、実装と乖離させない）。
- MVPで宣伝してよいプレミアム特典：**保管枠の拡大（20→200）・限定Mofi・プレミアム卵**（いずれも実装値と一致させる）。
